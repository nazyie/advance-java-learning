-- COLLECTION BEHAVIOUR IN THE JAVA

Collection in the java is the extension of the Iterable

Iterable -> Collection  -> Set
                        -> List
                        -> Queue

1. SET
- Collection that not allow the duplicate elements
- Unordered
- Eg: Card games with not duplicate values

2. LIST
- Allow the duplicate values
- Order is significant

3. Queue
- For FIFO
- Queue Live

4. Map
- Not extend Collection interface
- Key-value pairs
